Blog #6

Week 7

January 23, 2016

This week in DBC we moved on to JavaScript. During this week's assignments and readings I noticed that some of the things I saw in Ruby are similar to things I saw in JavaScript. One thing that stuck out was the Ruby hash and its similarities with the JavaScript Object.

In this week's blog I will take a look at both and show their similarities and there differences. How they are created and how they can be manipulated.

Lets first start with a basic example of both.

Ruby Hash:

grocery_list = {
"eggs" => 2,
"bread" => 3,
"milk" => 1
}

JavaScript Object:

var groceryList = {
eggs: 2,
bread: 3,
milk: 1,
};

Okay, let's look at the similarities and the differences. The similarities are that they are structured the same. They both contain the same keys and values. The differences are the way they are written and the way the key and values are declared. They both contain eggs with the quantity 2, bread with the quantity of 3 and milk with the quantity 1.

Starting from the top of each data structure we see that the ruby hash is declared by entering grocery_list with its key and value pairs written between curly brackets. The JavaScript object has to be declared with the keyword “var” first, then the objects name, groceryList, and then its key and value pairs store in between curly brackets.

The key and value pairs are declared differently in both data structures. In the ruby hash the key is a string and it is assigned a value using the hash rocket(=>). In the JavaScript object the key is written with a colon to assign the value. The values can be written as strings in quotes but the colon needs to be there. The colon sets the value just as the hash rocket sets the value in ruby. The semi colon at the end of the object states that it's completed.

So those are the similarities and differences between the Ruby hash and the JavaScript object. Ruby hashes can be set similar to the way that the JavaScript key and value pairs are set. You can declare a ruby hash as followed:

Ruby Hash:

grocery_list = {
eggs: 2,
bread: 3,
milk: 1
}

The way the key and value pairs are set up are the same as the JavaScript object above. The difference is that the keys are no longer strings they are now symbols.

Now lets look at a couple way to build Ruby Hashes and JavaScript objects.

Ruby Hash
grocery_list = {}

or

grocery_list = Hash.new


JavaScript Object

var groceryList = {};

or

var groceryList = new Object();

Above are examples of how a Hash and a Object can be constructed. They can both be set to equal curly brackets or written like the latter example of both. The examples set the ruby hash grocery_list to equal a blank hash and the JavaScript object to equal a blank object.

Now that we have a blank object and hash we can now start to add keys and values to them.

Ruby
grocery_list = {}

grocery_list['eggs'] = 2
grocery_list['bread'] = 3
grocery_list['milk'] = 1

Final hash will look like the hash we started with in the beginning.

grocery_list = {
"eggs" => 2,
"bread" => 3,
"milk" => 1
}

 JavaScript
var groceryList = {}

groceryList.eggs = 2;
groceryList.bread = 3;
groceryList.milk = 1;

The final object will look like the object we started with in the beginning.

var groceryList = {
eggs: 2,
bread: 3,
milk: 1,
};

So there you have it. This was just a quick look a 2 different data structures from 2 different programming languages, the JavaScript Object and the Ruby Hash. They look similar and they are both used the same by their respective languages. The difference is the way they are declared. From the examples we saw above Ruby lists the name of the hash and uses the hash rocket when declaring keys and values. JavaScript has to use the keyword var then the name. It then sets it key and value pairs by stating the key then a colon and then the value of that pair.

This is all I have this week. Thanks for reading.
