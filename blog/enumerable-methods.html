Blog #4

Week 5

January 10, 2016

For this week’s blog post I decided to talk about the enumerable method .reject. .reject returns an array for all elements for which the given block or instructions returns false. So if we added .reject to a array of numbers and in the block we tell it to reject the number 2 it will loop through each number in that array until 2 is found. Once 2 is found it .reject will remove it from the array. This example is set up below.

[1,2,3,4,5].reject { |i| i == 2 }

Results: [1,3,4,5]

So we have an array with the numbers 1,2,3,4,5. Then we add .reject. In the block you see i. i is the iterator. It takes the place of each number at the time that it is being iterated. In the block we’re giving instructions that say if i(the array number) equals 2, remove it from the array.

It’s a simple as that. If there are multiple twos in the array it will remove them all. If there are none it does nothing.

If no block or instructions are given the elements are just passed. An array is returned with the same elements in them.

[1,2,3,4,5].reject {|x|}

This will just loop through the array and return the number that is being passed through at the time. In order for .reject to work affectively, you have to pass instructions to the block. It could be anything from checking to see if x equals a number or if x is a even or odd number.
