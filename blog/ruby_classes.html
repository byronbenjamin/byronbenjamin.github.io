Blog #5

Week 6

January 17, 2016

Let's talk about Ruby classes. Ruby is an object-oriented programming language. Everything in Ruby is an object. In Ruby we use classes to help us create objects.

Classes are the formula, the blueprint, and the ingredients that make up an object. Each instance of a class is a new object. So if we create a class Cars, we can create new car objects using that class and its methods.

Class Car
def initialize(make, year, color)
@make = make
@year = year
@color = color
end

def make
@make
end

def year
@year
end

def color
@color
end
end

This is our Car Class. The first thing you have to do to make a Class is write Class and Name it. The first letter of the Class should be capitalized as seen with the Class Car. You then close it out with “end”, the same way you would do a method.

In the inside of the class we have 4 methods. The methods initialize, make, year, and color are created on the inside. The initialize method takes 3 arguments and set up the basic characteristics for your newly created Car object. The 3 arguments are then set to instance variables. Instance variable are variables that can be called anywhere within the class. To set an instance variable you have to use the "@" symbol followed by variable's name. Then you set that variable to equal something. Above we set @make to equal the make argument, we set @year to equal the year argument, and we set @color to equal the color argument. The purpose of using instance variables in this class is so that we can access them in the make, year, and color methods. If they were not set as instance variables the other methods in the class wouldn’t recognize them.

So now that we have our Class Car all set up we can start making new car objects. To do this we can do something like this:

car_one = Car.new("Chevy", 2016, "Blue")

We set a variable car_one to create a instance of Car with the make=Chevy, the year=2016, and color=Blue. The initialize method within the class then takes each variable and set them to a instance variable. The initialize method is called whenever a new Car object is created. The initialize method does not need to be called.

We now have a new object named car_one. Because it is an instance of the Car Class we can now call the methods created within that class. The methods created in the class returns the feature of that new car object when called, nothing too fancy. We would call the methods as followed:

car_one.make = Chevy
car_one.year = 2016
car_one.color = Blue

This was just a basic look at Ruby classes. Now that we have the Class Car we can easily create new Car objects over and over. We can create a whole fleet of cars. You can create more methods within the Class Car to help you manage that fleet. The possibilities are endless.
